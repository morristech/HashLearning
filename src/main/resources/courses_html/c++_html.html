Imperative Programming Concept:
Procedural programming is imperative programming in which the program
is built from one or more procedures (also known as subroutines or functions)
Consider a business man as a program, the business man has a lot of work to do,
so he has haired a good number of assistants.
The Business Man said:
Assistant 1 => handle financial affairs
Assistant 2 => handle consumers reports
Assistant 3 => keep track of sales
Assistant 4 => attend next meeting in my place at 2 o'clock and fill me in
As you can see, the Business Man gave each Assistant a task to do.
Here the Business Man is the program and his Assistants do the subroutines.
We define the aforementioned:
• Business Man's work as main function
• Assistants' work as subroutines or functions
• Routines description as statements
• The data they handle as variables
C/C++ Syntax:
C/C++ syntax is the rules we need to follow to write C/C++ codes.
• C/C++ main function, functions are represented as blocks enclosed in braces
{ }
• main function is the main block, from which program execution begins
• variables declared inside functions are called local variables
• variables declared outside functions are called global variables
• functions can accept data as arguments
• arguments are data enclosed in parentheses ( )
• arguments block is placed between method name and its braces { }
• argument declaration is identical to variable declaration
• statements are placed Inside functions blocks
• statements are terminated with a semicolon ;
• main function, functions and argument blocks can be left empty
• Anything after // or between /* and */ is a comment ignored by the compiler

Structs:
struct is a container that holds data / variables together in one place
To declare a struct named BMW, we do this:
struct BMW
{
}
Variables:
Declaring a variable requires specifying its data type as the compiler needs
to know whether our variable will hold numbers, text … etc
data-type variable_name;
data-type variable_name = value;
Note that we can assign a value to a field or leave it as it is!
Some C/C++ primitive data types:
int for numerical values like these: 0 , 1 , 2 , 4 , 16
double for numerical values like these: 0.1 , 1.5 , 2.0 , 4.6 , 16.3
char for characters like these: 'a', 'b', 'c', 'D', 'E', 'F', '0', '5', '%', '/'
Example, as BMW is a car, let's store something relative:
struct BMW
{
int number_of_seats;
int max_speed; // in meters
int car_has_started; // 0 = false / anything = true
}
int main(int argc, char** argv)
{
BMW my_car;
my_car.number_of_seats = 4; // family car
my_car.max_speed = 180000; // in meters
my_car.car_has_started = 0; // 0 = false / anything = true
start_car (my_car);
return 0;
}
Functions:
There are mainly 2 different types of routines:
1. Routines that return nothing
2. Routines that return a result
Remember:
• Declaring an argument is similar to declaring a field.
• A function can have zero or any number of arguments but only one return-
type or value.
To declare a function we write:
return-type function_name (data-type arg_1, data-type arg_2, …)
{
/*
statements
*/
}
Example:
Consider a function named multiply that accepts 2 ints and return their product:
int multiply (int num1, int num2)
{
int product = num1 * num2;
return product;
}
We can use it like this:
int max_speed = multiply(180, 1000); // pretty simple, isn't it?
What if we don't want to return anything?
We just omit return statement and change return-type to void, which means
nothing.
void multiply (int num1, int num2)
{
int product = num1 * num2;
}
Statements:
A statement could be:
• Declaration:
int product;
• Initialization:
product = 4 * 5;
• Declaration and initialization:
int product = 4 * 5;
• Struct creation:
BMW my_car;
• Method call:
meeting();
• Mixture:
my_car.max_speed = multiply(70, 6);
/* a dot ( . ) to access my_car's member */
Global Vs. Local Fields:
global variables are accessed from anywhere and destroyed when the
program ends, local variables can only be accessed from the function where the
were declared and they are destroyed once the function ends.
Arrays:
You can declare a variable like this:
int product;
You can also declare multiple ones like this:
int product;
int sum;
or this:
int product, sum;
But what if you have a lot of things to declare like a 1000 int?
The simplest solution is to use an int array ( int[] ):
int my_ints[];
To specify the number of elements in the array we initialize the array like this:
int my_ints_1[10]; // 10 elements
int my_ints_2[10] = { 0 } // 10 elements, all zeros
int my_ints_3[] = {1, 2, 4, 8}; // 4 elements 1, 2, 4, 8
array elements can be accessed by specifying their index.
array index starts from zero:
#1 element's index is [1 – 1] = [0]
#2 element's index is [2 – 1] = [1]
#26 element's index is [26 – 1] = [25]
and so on.
Example:
int product = multiply(my_ints_3[0], my_ints_3[1]); // multiply #1 and #2
elements of my_ints_3 array
Flow control:
Sometimes we need to repeat a certain statement a number of times or
execute a different one depending on a certain condition.
This calls for special blocks, loop blocks:
while (condition) {/* statements */}
do {/* statements */} while (condition)
for (initialization ; condition ; statement) {/* statements */}
Example:
int x = 0;
while (x < 10)
{
x++; // the same as (x = x + 1)
}
do
{
x--; // the same as (x = x - 1)
} while (x >= 0) // while x = 0 or greater
int size = 16;
int array[size];
for (int y = 0 ; y < size ; y++)
{
array[y] = y;
}
// all array elements now are equal to their index

Recursion:
Recursion in C/C++ means calling a function from inside its body causing an
infinite loop as each function call calls itself and each call to itself calls the same
function again!
Example:
void foo()
{
foo(); // endless loop
}
Recursion is rarely useful and should be avoided!
Passing Of Arguments:
You can pass argument to their function:
1. By value, as in previous examples
2. By reference, using something called pointers
Passing arguments by value means that passed-arguments are copied over to the
function, so only a copy of them goes to the function, no changes to the
arguments inside function's body affects the real arguments, just their copies.
Passing arguments by reference using pointers means that arguments aren't
passed to the function directly, rather their addresses are passed, the function then
loads the value from its address or stores a different one there.
Pointers:
What pointers really are?
A pointer is an address of a certain value in program / computer memory
A pointer is declared by suffixing the data type of the value it points to (has its
address) by *
A variable's address can be retrieved by prefixing variable name with &
A pointer's address can be retrieved by prefixing pointer name with &
Loading the value from pointer (the value this pointer points to) is done by
prefixing the pointer with *
Storing a value in a memory address a pointer points to is done by assigning that
value to *-prefixed pointer
Incrementing / decrementing a pointer returns next / previous memory address and
its value.
Example:
int sum = 128;
int* sum_pointer = &sum ; // sum_pointer = address of sum
int sum_copy = *sum_pointer; // sum_copy = 128
int** pointer_2_pointer = &sum_pointer; // pointer's address
int*** pointer_2_pointer_2_pointer = &pointer_2_pointer ;
char text[] = ''Hello, World!'';
text[5] == * (text + 5) == 'o'
text[x] == * (text + x)
You can think of a pointer as an array
You can think of a pointer to pointer as an array of arrays
You can think of a pointer to pointer to pointer as an array of arrays of arrays
This means that these expressions are valid and identical:
char* text = ''Hello, World!'';
char text[] = ''Hello, World!'';
char text[14] = ''Hello, World!''; // extra slot for terminating NULL
char** texts = { ''Hello, World!'', ''Okey'', ''Yo bodies'', NULL };
char* texts[] = { ''Hello, World!'', ''Okey'', ''Yo bodies'', NULL };
char texts[][] = { ''Hello, World!'', ''Okey'', ''Yo bodies'', NULL };

void add (int x, int y)
{
x = x + y;
}
void add_pointers (int* x, int* y)
{
*x = *x + *y;
}
int main (int argc, char* argv[])
{

add (a, b); // a = 5
add_pointers (&a, &b); // a = 15 now!

}
Memory Allocation & Reallocation:
C/C++ variables are stored in a storage called stack
stack isn't infinite, rather it's quite limited, you can't be sure that an array of a
million elements will be created without errors.
Luckily, we can store our data in much larger storage called heap.
Keep in mind that YOU are responsible for managing heap memory yourself.
Unlike stack, data stored in heap memory aren't destroyed when the program ends.
Leaving data in heap memory causes Memory Leak.
Memory Leak is when a program acquires memory from heap without returning it
so that other programs can use it.
Allocating is usually done by calling:
void* malloc (size_t size); // size in bytes
void* calloc (size_t n_elements, size_t element_size); // size = n_elements *
elements_size
Reallocating is done by:
void* realloc (void* memory, size_t new_size); // size = new_size
Freeing is done by:
int free (void* memory); // bye bye, don't leak!
All of them return a pointer to allocated memory.
Example:
int main (int argc, char* argv[])
{
// sizeof( type ) is an operator that returns type's size in bytes!
int* array = calloc ( 1000000, sizeof (int) ); // array of a million int!
// play with array
realloc (array, 10); // chibi!
free (array); // don't leak!
return 0;
}
